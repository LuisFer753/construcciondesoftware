Preguntas JS

1. ¿Por qué es buena práctica usar JavaScript para validar inputs antes de enviar al servidor?

Mejora la UX dando feedback inmediato sin recargas, reduce tráfico innecesario al servidor (ahorra ancho de banda y CPU), y permite validaciones complejas en cliente. Aunque no es 100% seguro, es eficiente para el 95% de casos normales.

2. ¿Cómo saltarte las validaciones de JavaScript?

Deshabilitar JS en navegador, usar DevTools para modificar DOM/valores, o enviar POST directo via curl/Postman burpando el request. JS solo valida cliente-side.

3. Si se puede saltar, ¿por qué usarlo?

UX superior (feedback instantáneo), menor carga servidor (solo datos válidos llegan), y capa defensa múltiple: JS=UX+optimización, servidor=seguridad obligatoria. Como en tus proyectos CUDA: optimiza cliente, valida server.
