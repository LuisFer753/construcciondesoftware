Preguntas sobre CSS

1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Recomiendo usar !important solo en casos muy puntuales, cuando no exista otra alternativa razonable, porque rompe la cascada de CSS y dificulta el mantenimiento del código. Un abuso de !important obliga luego a escribir reglas cada vez más específicas y hace más complejo depurar estilos.

2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Debe elegirse con cuidado porque una imagen muy pesada aumenta el tamaño de descarga y empeora el tiempo de carga del sitio, sobre todo en conexiones lentas. Además, un fondo mal elegido puede afectar la legibilidad del texto si no hay suficiente contraste o si distrae demasiado al usuario.

3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de una propiedad de estilo entre %, px y pt?

Recomiendo usar px para tamaños que deben ser muy precisos en pantalla (como bordes o iconos), ya que se comporta como unidad de píxel del dispositivo. Para diseños responsivos, es mejor usar unidades relativas como % para que el contenido se adapte a distintos tamaños de pantalla. Las unidades en pt se reservan más para contextos de impresión, no tanto para interfaces web modernas.

4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Un CSS minimizado elimina espacios, saltos de línea y tabulaciones, reduciendo el tamaño del archivo. Al ser más pequeño, se transfiere más rápido desde el servidor al navegador, lo que mejora el tiempo de carga y la experiencia del usuario.
